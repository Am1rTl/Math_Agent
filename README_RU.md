# MATH-AGENT-VL 🧮🤖

[**English Version**](README.md)

MATH-AGENT-VL — это эталонная реализация пайплайна **Визуально-Языкового Математического Агента (Vision-Language Math Agent)**. Проект разработан для решения сложных многоступенчатых математических задач и задач на логику путем объединения структурированного планирования (Tree of Thoughts) с выполнением действий с помощью инструментов (ReAct).

## 🚀 Особенности

- **Многоступенчатые рассуждения:** Решает сложные математические задачи, разбивая их на подзадачи с помощью когнитивной архитектуры.
- **Архитектура ToT-over-ReAct:** Использует древовидное планирование (Tree-of-Thought, ToT) для построения надежного плана и ReAct (Reason + Act) для его выполнения.
- **Интеграция инструментов:** Встроенная поддержка Python-песочницы (PAL), SymPy для символьной математики и WolframAlpha для запросов к внешним базам знаний.
- **Формальная верификация:** Включает этап проверки (верификатор и критик) для математического доказательства и контроля шагов перед выдачей финального ответа.
- **Веб-интерфейс:** Веб-приложение на базе Flask для управления задачами, визуализации процесса решения и наблюдения за ходом мыслей агента в реальном времени.

## 📂 Структура репозитория

Проект был организован так, чтобы всё было на своих местах:

- **`main.py`** — Главная точка входа для запуска пайплайна MATH-AGENT-VL.
- **`web/`** — Содержит Flask веб-приложение, статические файлы и шаблоны для пользовательского интерфейса.
- **`agents/`** — Альтернативные реализации и автономные скрипты агентов (например, Qwen, DeepSeek, экспериментальные скрипты).
- **`docs/`** — Документация. Обязательно загляните в [`how-it-works.md`](docs/how-it-works.md) для глубокого понимания архитектуры проекта.
- **`logs/`** — Логи выполнения агентов и веб-сервера.
- **`results/`** — Выходные JSON-файлы с результатами предыдущих запусков агента.
- **`legacy/`** — Старые тестовые файлы, проверки работоспособности (health checks) и данные больших промптов, сохраненные для истории.

## 🧠 Основная архитектура

Пайплайн работает в четыре основных этапа:

1. **Формализатор ввода (Input Formalizer):** Преобразует неструктурированный текст задачи в структурированный объект `ProblemObject` (извлекает цели, ограничения и сущности).
2. **Планировщик (Tree-of-Thought Planner):** Исследует 2-3 различные стратегии решения задачи и выбирает лучшую на основе эвристической оценки.
3. **Исполнитель (ReAct Executor):** Выполняет выбранный план шаг за шагом, используя цикл `Рассуждение (Thought)`, `Действие (Action)` и `Наблюдение (Observation)`. Активно использует инструменты, такие как выполнение Python-кода и SymPy.
4. **Верификатор и Критик (Verifier & Critic):** Процесс формальной верификации, обеспечивающий математическую обоснованность трассы рассуждений и финального ответа.

Для более подробного разбора архитектуры прочитайте [Документацию](docs/how-it-works.md).

## 🛠️ Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone <repository_url>
   cd math-agent-vl
   ```

2. **Установите зависимости:**
   Убедитесь, что у вас установлен Python версии 3.9+.
   ```bash
   pip install -r requirements.txt
   ```
   *(Примечание: требуются SymPy и другие инструменты. Если файла `requirements.txt` нет, можно установить вручную: `pip install sympy flask requests openai google-generativeai`)*

3. **Настройте переменные окружения:**
   Вам понадобятся API-ключи для работы с LLM и внешними инструментами.
   ```bash
   export GEMINI_API_KEY="ваш_ключ_gemini"
   export OPENROUTER_API_KEY="ваш_ключ_openrouter"
   export WOLFRAM_API_KEY="ваш_ключ_wolfram"
   ```

## 🖥️ Использование

### Командная строка
Вы можете запустить основной пайплайн напрямую из консоли:
```bash
python main.py
```

### Веб-интерфейс
Для запуска веб-сервера Flask:
```bash
python web/app.py
```
После этого откройте браузер и перейдите по адресу `http://localhost:5051`. В веб-интерфейсе можно создавать задачи, настраивать API-ключи и наблюдать за процессом выполнения агента в прямом эфире.
